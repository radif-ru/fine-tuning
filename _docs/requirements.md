# Требования

## Функциональные требования (FR)

### FR-1: Загрузка моделей

Система должна поддерживать загрузку любых causal language models из HuggingFace Hub по имени или из локального пути.

**Критерии приемки:**
- Поддержка моделей с `AutoModelForCausalLM`
- Автоматическое определение токенизатора
- Возможность указать trust_remote_code
- Graceful handling ошибок загрузки

### FR-2: LoRA адаптация

Система должна поддерживать параметро-эффективное дообучение через LoRA (Low-Rank Adaptation).

**Критерии приемки:**
- Конфигурация rank, alpha, dropout через .env
- Настраиваемые target modules
- Поддержка различных task_type
- Сохранение и загрузка адаптеров

### FR-3: Загрузка данных

Система должна поддерживать загрузку данных из различных источников и форматов.

**Критерии приемки:**
- JSON, JSONL, CSV файлы
- HuggingFace datasets
- Настраиваемые column mapping
- Валидация формата данных

### FR-4: Токенизация

Система должна выполнять токенизацию данных с поддержкой truncation и padding.

**Критерии приемки:**
- Настраиваемый max_seq_length
- Поддержка instruction/response формата
- Добавление специальных токенов

### FR-5: Обучение

Система должна выполнять обучение с логированием и сохранением чекпоинтов.

**Критерии приемки:**
- Интеграция с transformers Trainer
- Сохранение чекпоинтов по steps/epochs
- Early stopping
- Resume from checkpoint

### FR-6: Инференс

Система должна поддерживать генерацию текста дообученными моделями.

**Критерии приемки:**
- CLI для инференса
- Интерактивный режим
- Настраиваемые параметры генерации
- Batch inference

### FR-7: Конфигурация

Все параметры должны настраиваться через .env файл.

**Критерии приемки:**
- pydantic-settings для валидации
- Понятные сообщения об ошибках
- Значения по умолчанию
- Type hints

### FR-8: Логирование

Структурированное логирование всех операций.

**Критерии приемки:**
- Уровни DEBUG, INFO, WARNING, ERROR
- Логирование в файл и консоль
- Настраиваемый формат
- Никаких print

### FR-9: Мониторинг

Интеграция с внешними системами мониторинга.

**Критерии приемки:**
- Weights & Biases
- TensorBoard
- Логирование метрик

### FR-10: Расширяемость

Возможность расширения для новых моделей и форматов данных.

**Критерии приемки:**
- Модульная архитектура
- Интерфейсы для плагинов
- Примеры расширений

## Нефункциональные требования (NFR)

### NFR-1: Производительность

- Поддержка mixed precision (fp16/bf16)
- Gradient checkpointing для экономии памяти
- Gradient accumulation

### NFR-2: Надежность

- Обработка OOM ошибок
- Resume from checkpoint
- Валидация входных данных

### NFR-3: Масштабируемость

- Multi-GPU поддержка через accelerate
- Distributed training

### NFR-4: Удобство использования

- Чистый CLI interface
- Понятные сообщения об ошибках
- Примеры конфигураций

### NFR-5: Тестируемость

- Unit-тесты для всех компонентов
- Integration тесты
- Покрытие > 70%

## Ограничения (CON)

### CON-1: Типы моделей

Поддерживаются только causal language models (GPT-like). Encoder-only (BERT-like) и encoder-decoder (T5-like) — вне скоупа.

### CON-2: Методы обучения

Поддерживается только LoRA. Full fine-tuning и другие PEFT методы (IA³, Prompt Tuning) — в roadmap.

### CON-3: Квантизация

В MVP поддерживается только обычные веса (float16/float32). QLoRA (4-bit) — в roadmap.

### CON-4: Платформа

Приоритет — Linux. macOS и Windows — best effort.
