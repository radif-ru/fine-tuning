# Спринт 00. Bootstrap

- **Источник:** Инициализация проекта fine-tuning LLM с LoRA
- **Ветка:** `main` (инфраструктурный спринт; feature-ветка не заводится)
- **Открыт:** 2026-04-30
- **Закрыт:** 2026-04-30

## 1. Цель спринта

Подготовить **полноценный архитектурный фундамент** под фреймворк дообучения LLM: документация (`_docs/`), доска (`_board/`), скелет каталогов (`app/`, `tests/`, `configs/`), корневые файлы (`README.md`, `.gitignore`, `.env.example`, `requirements.txt`, `pyproject.toml`).

Никакого продуктового кода не пишется — только заготовки и пустые пакеты с `__init__.py`. Логика загрузки моделей, LoRA, обучения и инференса появится в последующих спринтах по плану из `_docs/roadmap.md`.

## 2. Скоуп и non-goals

### В скоупе

- Создание `.gitignore`, `.env.example`, `requirements.txt`, `pyproject.toml`, `README.md`
- Полная документация `_docs/` (10+ файлов: README, requirements, architecture, configuration, training, inference, testing, instructions, project-structure, roadmap, current-state)
- Полная доска `_board/`: `README.md`, `plan.md`, `process.md`, `progress.txt`, `sprints/00-bootstrap.md` (этот файл)
- Создание `CLAUDE.md` в корне проекта
- Скелет `app/` со всеми пустыми пакетами и `__init__.py`
- Скелет `tests/` с `conftest.py` и пустыми пакетами
- Создание runtime-директорий (`data/`, `logs/`, `checkpoints/`, `outputs/`) с `.gitkeep`

### Вне скоупа (non-goals)

- Реализация конфигурации (`Settings`), логирования, исключений — в Спринте 01
- Реализация загрузки моделей, LoRA, реестра моделей — в Спринте 02
- Реализация загрузки данных, форматирования, токенизации — в Спринте 03
- Реализация тренировочного цикла — в Спринте 04
- Реализация инференса — в Спринте 05
- Реальный запуск тренировки, прогон кода — в Спринтах 01-05

## 4. Решения по архитектуре, принятые в этом спринте

| Решение | Ссылка на документ | Краткое обоснование |
|---------|---------------------|---------------------|
| **Язык документации — русский** | `_docs/instructions.md` §0 | Запрос пользователя; технические идентификаторы — латиницей |

| **PEFT + LoRA** | `_docs/architecture.md` §2 | Стандарт де-факто для parameter-efficient fine-tuning |
| **Transformers Trainer** | `_docs/training.md` | Надёжная база, много опций, хорошая документация |
| **Pytest для тестирования** | `_docs/testing.md` | Стандарт для Python, хорошая экосистема плагинов |
| **Ruff для линтинга** | `_docs/instructions.md` §2 | Быстрый, заменяет flake8 + black + isort |
